# Project Rules

## 資料規則

- 優先使用 TWSE、TPEx、MOPS 等官方來源。
- 不可猜測 API 欄位、資料單位或更新時間。
- 明確標示股、張、元、千元、百分比與契約口數。
- 原始資料與清理後資料分開保存。
- 回測必須防止前視偏誤、存活者偏誤與資料洩漏。
- 必須納入手續費、交易稅、滑價與流動性限制。

## 程式規則

- Python 遵守 PEP 8。
- 函式與類別需有 docstring。
- 金融邏輯使用繁體中文註解。
- 資料、因子、策略、回測與報告分離。
- 設定值集中於 `config/`。
- API 呼叫需包含 timeout、retry、錯誤處理與 logging。
- 不提交 API Key、密碼、憑證或客戶資料。

## 測試規則

至少測試：

- 正常資料
- 空資料
- 缺欄位
- 非交易日
- API 失敗
- 單位轉換
- 重複資料
- 極端值
- 多空方向符號

## 回測標準輸出

- 交易次數
- 勝率
- 平均獲利
- 平均虧損
- 期望值
- Profit Factor
- 累積報酬
- 最大回撤
- Sharpe Ratio
- Sortino Ratio
- 交易成本
- 多方與空方分開結果

## Git Commit 規範

- `feat:` 新功能
- `fix:` 修正錯誤
- `refactor:` 重構
- `docs:` 文件
- `test:` 測試
- `data:` 資料處理
- `chore:` 維護
